# Anamnesis Classifier — классификация анамнезов в группы фитнес-ограничений

Классификатор текста жалоб пациента (анамнез) → **группа фитнес-ограничений**: какой тип
нагрузки требует осторожности при подборе тренировочной программы.

## Проблема и продукт

- **Проблема.** Перед стартом тренировок фитнес-сервису/тренеру нужно понять противопоказания
  клиента по его анамнезу. Ручной скрининг медленный и требует мед. квалификации.
- **Целевая аудитория.** Онлайн фитнес-сервисы, студии, персональные тренеры.
- **MVP.** Модель «текст жалоб → группа ограничения» + inference-скрипт.
- **Импакт.** Автоматический первичный скрининг: какие нагрузки требуют осторожности.

## Группы ограничений (9 классов)

Метка в данных — код МКБ-10; мы маппим его в группу по **типу ограничения нагрузки**
(а не просто по системе органов). Маппинг — в [`src/mapping.py`](src/mapping.py),
машинная копия — [`data/label_mapping.json`](data/label_mapping.json).

| Группа | Суть ограничения |
|---|---|
| `spine` | осевая/компрессионная нагрузка (M-дорсопатии + G54 радикулопатия) |
| `joints` | ударная нагрузка на суставы (артриты/артрозы) |
| `cardio` | контроль ЧСС/давления (болезни системы кровообращения) |
| `metabolic` | интенсивность/режим (ожирение, диабет, щитовидка) |
| `respiratory` | контроль дыхания (хронич.: астма/ХОБЛ) |
| `neuro_balance` | баланс/координация/падения (+ вестибулярные) |
| `core_visceral` | внутрибрюшное давление (ЖКТ + урогенитальные + миома) |
| `acute` | острая инфекция → временный отвод от тренировок |
| `none` | нет значимых ограничений |

**Спорные коды требуют медицинской валидации** (см. комментарии в `src/mapping.py`):
`G54→spine`, `G43/G44→neuro_balance`, `D50→none` (кандидат в metabolic),
`J30/J35→none`.

## Структура

```
src/        mapping, data, preprocess, metrics, models/ (tfidf, e5_mlp, lora)
scripts/    export_mapping.py, run_tfidf_baseline.py
notebooks/  anamnesis_pipeline.ipynb  (E5/LoRA, гоняется в Colab на GPU)
tests/      test_mapping.py  (покрытие кодов, согласованность сумм)
data/       train/dev/test jsonl + label_mapping.json
reports/    метрики и матрицы ошибок
```

## Запуск локально

```bash
py -3 -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt   # Windows
py -3 -m pytest -q                       # тесты
py -3 scripts/export_mapping.py          # сгенерировать json + распределение
py -3 scripts/run_tfidf_baseline.py      # baseline на dev
```

Обучение трансформеров (E5, LoRA) — в Google Colab на GPU (`requirements-train.txt`):
на локальной машине NVIDIA нет, CUDA недоступна.
